# 幻觉分类体系（Hallucination Taxonomy）

> 系统化分类 LLM 生成内容中不同类型错误和虚构信息的框架，包括内在幻觉 vs 外在幻觉、事实性幻觉 vs 忠实性幻觉等核心维度。

## 定义

幻觉（Hallucination）在 LLM 语境中指模型生成的与事实、源文本或逻辑不符的内容。系统化的分类有助于设计针对性的评测方法。

**主要分类维度**：

**按来源分**：
- **内在幻觉（Intrinsic Hallucination）**：生成内容与输入源文本（如 RAG 检索结果）矛盾
- **外在幻觉（Extrinsic Hallucination）**：生成内容无法从源文本中验证，可能是虚构或无关信息

**按错误类型分**：
- **事实性幻觉（Factual Hallucination）**：与客观世界事实冲突（错误的人物/地点/时间/数字）
- **忠实性幻觉（Faithfulness Hallucination）**：对给定上下文不忠实（摘要添加原文没有的内容）
- **推理幻觉**：逻辑推理链中出现的错误步骤

**按严重程度分**：
- 完全虚构（fabrication）：无中生有
- 错误细节（confabulation）：真实事件的错误细节
- 合理化编造（plausible but false）：听起来合理但实际错误

## 重要性（在 LLM 评测中）

幻觉分类对评测方法设计至关重要：

1. **评测工具选择**：事实性幻觉需要外部知识库验证；忠实性幻觉只需与输入文本比对，评测难度不同
2. **任务特定评测**：摘要任务主要关注忠实性幻觉；QA 任务主要关注事实性幻觉
3. **RAG 专项评测**：RAG 系统中内在幻觉（忽视检索结果）和外在幻觉（凭空编造）需分别评测
4. **严重性权重**：高风险应用（医疗/法律）应对不同幻觉类型赋予不同严重性权重

## 主要方法/实现

**主要幻觉评测基准**：
- **TruthfulQA**（Lin et al., 2021）：测试模型在容易产生"常识性"幻觉的问题上的表现
- **FActScoring**（Min et al., 2023）：原子事实分解后逐条验证
- **HaluEval**：在多个 NLP 任务上系统评测幻觉率
- **FactScore**：生成内容的每个原子命题通过外部文档验证

**FActScore 流程**：
```
生成文本 → 原子事实分解 → 
每个原子事实独立验证（搜索引擎/知识库） → 
幻觉率 = 无法验证的原子事实比例
```

**忠实性评测（NLI-based）**：用自然语言推理（NLI）模型判断生成内容是否蕴含在源文档中，如 SummaC 框架。

## 局限与挑战

- **验证基础设施依赖**：事实性幻觉验证需要可靠的外部知识源，互联网搜索本身也可能不准确
- **原子分解的主观性**：将生成文本分解为原子命题本身就引入主观性
- **幻觉定义争议**：模型表达不确定性时的"可能"陈述是否算幻觉？语用层面的定义不清晰
- **知识截止日期**：模型知识截止后的真实事件，对模型而言是"外在幻觉"但对用户是关键信息缺失
- **幻觉 vs 创意写作**：在创意任务中，虚构内容是期望行为，分类需考虑任务意图
