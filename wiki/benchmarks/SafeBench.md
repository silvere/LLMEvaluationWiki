# SafeBench

> 多任务安全对齐评测基准，系统评估 LLM 在多维度安全场景下的行为。

## 概述

SafeBench 是 2024 年提出的 LLM 安全对齐综合评测基准，旨在通过多任务、多场景的统一评测框架系统性地评估大型语言模型的安全行为。与专注于有害内容拒绝的基准（如 SALAD-Bench）不同，SafeBench 覆盖了更广泛的安全维度，包括有害内容生成、虚假信息传播、隐私泄露、公平性偏见、越权行为等多类风险。

SafeBench 的设计将安全评测分解为多个子任务，每个子任务对应一类具体的安全风险，并为每个子任务设计了专门的评测指标和标注规范。这种多任务设计使研究者能够精确定位模型在哪些安全维度上存在不足，而非仅得到一个笼统的安全分数。

基准包含对**多模态安全**的初步支持，考察包含图像内容的有害请求，是安全评测领域向多模态扩展的早期探索之一。

## 任务格式

- **任务类型**：有害内容生成、虚假信息、隐私泄露、偏见歧视、越权操作、多模态安全等多个子任务
- **输入**：文本问题或图文组合（多模态子任务）
- **输出**：LLM 回应文本
- **规模**：数千个测试实例，覆盖 10+ 安全子任务类别
- **评判**：自动评分 + 人工标注一致性验证

## 主要指标

- **子任务安全率**：每个安全子任务的模型合规比例
- **综合安全得分**：跨子任务的加权平均安全性
- **False Refusal Rate**（误拒率）：模型对合法请求的错误拒绝比例（衡量过度对齐）
- 多模态子任务另设视觉-文本安全联合评分

## 局限性

- 多任务设计导致子任务间比较基准不统一，总体得分解释需谨慎
- 多模态安全部分规模较小，结论代表性有限
- 安全标准随社会规范演变，评测集内容需定期更新
