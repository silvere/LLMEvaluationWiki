# 评测成本分析

## 概述

LLM 评测的成本往往被低估。评测不仅有直接的经济成本（计算费用、API 费用、人工费用），还有时间成本和机会成本。理解不同评测方式的成本构成，有助于从业者根据资源约束做出合理的评测策略选择。

## 成本维度

评测成本可从三个维度衡量：
- **经济成本**：直接货币支出（API 费用、GPU 租用、人工费用）
- **时间成本**：从开始评测到获得结果所需的时间
- **工程成本**：建立评测基础设施、维护评测管道所需的工程投入

## 主要评测方式的成本比较

### 标准自动评测（lm-evaluation-harness 类）

**被评测对象：开源模型**
- 经济成本：GPU 算力，7B 模型评测一套基准约需 $1-$10（A100 GPU 小时计）
- 时间成本：几小时内完成
- 工程成本：初始配置约需 1-2 天，后续维护较低

**被评测对象：API 模型（GPT-4 等）**
- 经济成本：MMLU（14K 题）× GPT-4 Turbo 约 $50-$100；完整 HELM 评测可达 $1000+
- 时间成本：取决于 API 速率限制，通常几小时到几天
- 工程成本：较低，主要是 API 集成

### LLM-as-Judge 评测（AlpacaEval 类）

**成本结构**：
- 生成被评测模型的回答：视模型和问题数量而定
- GPT-4 评判：AlpacaEval（805 题×2次调用）≈ $10-$30
- Arena-Hard（500 题）≈ $5-$20

**时间成本**：通常几小时内完成（受 API 速率限制）

### 人工评测

**众包评测（MTurk/Prolific）**：
- 单条评测费用：$0.10-$0.50（取决于任务复杂度和标注要求）
- 获得 1000 条高质量标注：约 $200-$600
- 时间成本：数天到一周（取决于任务发布到完成的时间）

**专业标注者**：
- 单条评测费用：$1-$10（有专业背景的评测者）
- 专业红队测试：$50-$200/小时（安全专家）

**Chatbot Arena 模式（众包免费投票）**：
- 经济成本：主要是服务器运营成本（每条投票约 $0.01 以下）
- 时间成本：获得统计显著的排名需要数周到数月

### 代理评测（SWE-bench 类）

代理评测是目前成本最高的评测类型：
- 每个任务需要运行实际代码（沙箱环境），约需 10-60 分钟
- SWE-bench（2294 题）完整评测：GPU + 时间成本约 $500-$2000
- 顶级模型提交 SWE-bench 的总成本（API + 基础设施）可达 $10,000+

## 成本-效益权衡框架

不同场景下的推荐策略：

| 预算 | 场景 | 推荐方案 |
|-----|-----|---------|
| < $10 | 快速原型验证 | 小样本 LLM-as-Judge（100-200条）|
| $10-$100 | 开发迭代评测 | AlpacaEval + 1-2个客观基准 |
| $100-$1000 | 模型发布前评测 | 完整排行榜套件 + 小规模人工抽样 |
| > $1000 | 生产级评测 | 多维度系统评测 + 人工红队测试 |

## 隐性成本：评测的工程债务

一个常被忽视的成本是**评测基础设施的维护成本**：
- 评测代码需要随模型 API 变化而更新
- 数据集版本管理（防止使用了被污染的版本）
- 评测结果的存储和对比系统建设

对于持续迭代的 AI 产品团队，建立完善的评测基础设施的一次性工程成本可能相当于数周的工程师工时。

## 趋势

随着 LLM 评测框架（DeepEval、LangSmith、Braintrust 等）的成熟，评测的工程成本在下降；但随着模型本身变得更强，需要更复杂（也更昂贵）的评测任务才能体现差异，导致顶尖模型的评测成本持续上升。
