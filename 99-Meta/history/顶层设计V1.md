# LLM 评测 Wiki — 顶层设计 (P10 战略草案)

## Context (为什么做这件事)

中文世界目前缺一个**系统性 + 批判性 + 实战性**三位一体的大模型评测知识库。现有资源散落在：
- 英文侧 (Papers with Code, HELM, LMSYS blog, Stanford AI Index) — 中文读者门槛高
- 中文自媒体 — 碎片化、追热点、缺深度
- 学术论文 — 颗粒度太细、缺全景

本项目目标：做一个**"评测界的漫步华尔街"** — 让从业者看懂榜单、识别 BS、做出决策。最终形态：Obsidian 文件夹源 + 对外静态站点，持续迭代，形成个人/团队在 LLM 评测赛道的心智占位。

---

## 九大关注点 (9 个抓手)

### 1. 顶层定位 (Positioning)
- **一句话价值主张**: "看懂 LLM 评测的中文第一站"
- **受众分层**:
  - P0: 一线研究者 / 评测工程师 (深度 + 可操作)
  - P1: 产品经理 / 决策者 (选型参考 + 榜单解读)
  - P2: 学习者 / 媒体 / 投资人 (科普 + 批判视角)
- **差异化**:
  - 中文原生 (非翻译)
  - 敢于批判 (数据污染、刷榜、Goodhart 现象指名道姓)
  - 实战导向 (附 lm-eval-harness / OpenCompass 上手教程)
  - 持续更新 (季度快照, 榜单月报)

### 3. 原子笔记规范 (Zettel + MOC)
- 每个 benchmark / 概念 = 一个原子笔记 (≤1500 字)
- 模板统一字段：`定义 / 提出者 / 数据规模 / 评测方法 / 代表模型表现 / 局限与争议 / 参考文献`
- 双链贯穿 (`[[MMLU]] 衍生出 [[MMLU-Pro]]`)
- 标签分层 `#benchmark #method #critique #tool #industry/code`
- Dataview 动态汇总：所有 benchmark 按年份/维度/语言聚合成表

| 元素 | 方案 |

### 5. 技术栈 (Obsidian → 站点)

**锁定方案**: Obsidian (源) + **Quartz v4** (静态站点生成) + Cloudflare Pages (部署)

- Quartz 原生吃 Obsidian 双链、图谱、Callout、Wikilink
- Cloudflare Pages 免费 + 国内可访问
- 保持 Obsidian 为唯一真相源, 无需二次写作

**已放弃的备选**:
- Docusaurus: 工程化强但双链支持弱
- MkDocs Material: 简洁但不原生吃 Obsidian
- Obsidian Publish: 贵 ($10/月) + 样式不可控

### 6. 内容治理
- **引用规范**: 论文用 `[作者 年份]` + arXiv ID, 榜单附快照日期
- **版本快照**: 每季度 git tag (LLM 领域半衰期短, 需留存历史视角)
- **批判模块强制**: 每个 benchmark 卡片必带 "局限/争议" 段落
- **更新节奏**: 每周 1 篇深度 + 榜单月报 + 季度综述

### 7. 贡献与协作
- **双写架构**: Obsidian 主站 (私) + GitHub 仓库 (公) — 用 `git` 同步
- 社区贡献走 PR (GitHub) + Issue 讨论
- Meta 目录含维护者手册, 降低新贡献者上手成本

### 8. 发布与运营
- **域名**: `llm-eval.wiki` 或 `eval-ai.cn` (建议先抢注)
- **SEO**: 每页 meta description, sitemap.xml, structured data
- **搜索**: Quartz 内置 flexsearch / 升级 Algolia DocSearch
- **分析**: Umami (隐私友好, 自托管)
- **RSS**: 月报 + 新增深度文 推送
- **双语策略**: v1 中文 only, v2 英文 (用 Claude 批量翻译 + 人审)

### 9. Roadmap (3 Sprint, 6 周)

| Sprint | 周期 | 交付 |
|--------|------|------|
| S1 "骨架" | W1-W2 | IA 定稿 + VI 定稿 + 10 个核心 benchmark 原子笔记 + Quartz 搭建 + 域名备案 |
| S2 "血肉" | W3-W4 | Benchmark 全景图 (30+) + 方法论 + 工具链 + 首页 MOC + 评论/搜索 |
| S3 "灵魂" | W5-W6 | 批判章节 + 行业应用 + 运营文案 + v0.1 公开发布 |
