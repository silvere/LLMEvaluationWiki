# Evaluating Large Language Models Trained on Code

> 介绍 Codex 模型与 HumanEval 基准，以 pass@k 指标系统评估大语言模型的 Python 代码生成能力。

## 核心主张
- [CLAIM]: 论文提出 HumanEval 基准，包含 164 道手工编写的 Python 编程题，每题附单元测试，通过运行测试而非字符串匹配来判断正误。
- [CLAIM]: 引入 pass@k 指标：对每道题生成 k 个样本，只要有一个通过所有测试则算成功，反映模型的最优采样能力。
- [CLAIM]: Codex（基于 GPT-3 微调，在 GitHub 代码上训练）的 pass@1 约为 28.8%，pass@100 约为 72.3%，优于未经代码微调的 GPT-3（接近 0%）。
- [CLAIM]: 模型在生成正确代码方面存在明显的"长尾"问题：简单题目表现良好，复杂多步骤推理题目准确率骤降。

## 方法论摘要

HumanEval 题目由 OpenAI 研究人员手工编写，每题包含函数签名、文档字符串（docstring）和单元测试（平均每题 7.7 个测试）。评测时，模型仅接收函数签名和 docstring 作为提示，需补全函数体。pass@k 的无偏估计公式避免了直接计算 k 次采样的方差问题。

## 数据与结果

- HumanEval 规模：164 道题目
- Codex-12B pass@1：28.8%
- Codex-12B pass@100：72.3%
- GPT-3（无代码微调）pass@1：~0%
- GPT-J（6B 开源模型）pass@1：11.4%
- 人类程序员估计：~90%（非正式基准）

## 局限与争议

- HumanEval 题目全部为 Python，不覆盖多语言编程场景。
- pass@k 随 k 增大会趋近 100%，对实际使用场景的参考意义有限。
- 单元测试覆盖率有限，存在"通过测试但代码有误"的情况（测试不充分）。
- 发布后被广泛使用，导致基准污染问题逐渐显现。
