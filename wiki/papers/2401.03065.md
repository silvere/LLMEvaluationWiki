# CRUXEval: A Benchmark for Code Reasoning, Understanding and Execution

> 提出 CRUXEval，包含 800 道 Python 函数题，测试模型的代码推理能力：给定函数和输入预测输出（输出预测），或给定函数和输出反推输入（输入预测）。

## 核心主张
- [CLAIM]: CRUXEval 区别于现有代码生成基准，专注于代码理解和执行推理：给定一段 Python 函数和输入，要求模型预测输出（CRUXEval-O），或给定函数和输出，要求推断可能的输入（CRUXEval-I）。
- [CLAIM]: 在 CRUXEval-O 上，GPT-4 的 pass@1 约为 58%，代码专用模型 Code Llama（34B）约为 38%，说明代码推理与代码生成能力并不完全相关。
- [CLAIM]: 思维链（CoT）提示在 CRUXEval 上提升幅度约为 5-15%，但对某些模型反而有轻微下降，说明 CoT 对代码执行追踪任务的效果不如数学推理稳定。
- [CLAIM]: CRUXEval-I（输入预测）比 CRUXEval-O（输出预测）更难，所有模型的准确率均更低，反映反向代码推理的额外难度。

## 方法论摘要

800 道题目通过程序合成和过滤生成：先用 Code Llama 生成 Python 函数，执行得到真实输入/输出对，过滤掉过于简单或异常的函数。评测使用 pass@1 指标，比较精确字符串匹配。

## 数据与结果

- 题目数：800（CRUXEval-I 和 CRUXEval-O 各 800）
- GPT-4 CRUXEval-O pass@1：约 58%
- Code Llama 34B：约 38%（CRUXEval-O）
- CoT 提升：GPT-4 约 +8%（具体数字依设置）
- 人类程序员估计：约 85%+（非正式）

## 局限与争议

- 题目由程序自动生成，可能存在偏向特定代码模式（如短函数）的分布偏差。
- 精确字符串匹配可能过于严格（如浮点输出可能有格式差异）。
- 仅覆盖 Python，代码推理能力的语言泛化未验证。
