# 自动评测与人工评测的对比

> 系统比较基于指标/模型的自动评测方法与人类判断的优缺点，两者在 LLM 评测中扮演不同但互补的角色。

## 定义

**自动评测（Automated Evaluation）**：使用确定性指标（BLEU/F1/Pass@k）或 LLM 裁判（GPT-4/LLM-as-judge）在无需人工介入的情况下评测模型输出质量。

**人工评测（Human Evaluation）**：招募人类（专家或众包）对模型输出进行直接质量判断，被视为评测的黄金标准。

两者并非简单替代关系：人工评测是校准自动评测的标准，自动评测是人工评测的规模化代理。

## 重要性（在 LLM 评测中）

理解两种方法的权衡对评测设计至关重要：

1. **效度 vs 成本权衡**：人工评测效度更高但成本数十倍于自动评测，研究预算约束下需要明智选择
2. **自动化的验证需求**：任何新提出的自动评测方法都需要在人工评测上验证相关性
3. **指标选择依据**：不同任务类型应选择不同的自动化指标（代码选 Pass@k，翻译选 BLEU/COMET）

## 主要方法/实现

**对比维度分析**：

**速度与成本**：
- 自动评测：毫秒级，成本接近零（指标）或极低（LLM 裁判）
- 人工评测：天/周级，每样本 $0.05-$5（众包）到数十美元（专家）

**可靠性（信度）**：
- 自动评测：完全可复现，无随机性（确定性指标）
- 人工评测：存在 IAA 问题，但真实反映人类判断的分布

**有效性（效度）**：
- 自动评测：与人类判断相关性各异（BLEU 相关性低，LLM-as-judge 相关性较高）
- 人工评测：被视为黄金标准，但人类偏好本身也可能有偏差（谄媚效应）

**覆盖面**：
- 自动评测：可覆盖数万个样本
- 人工评测：通常 200-1000 个样本

**适用任务推荐**：
```
选择题/填空题 → 自动 EM/F1（高效度）
代码生成 → 自动 Pass@k（客观执行结果）
机器翻译 → 自动 COMET + 人工抽样验证
摘要 → LLM-as-judge + 人工校准
对话/开放生成 → LLM-as-judge + 定期人工评测
安全评测 → 人工专家 + 自动过滤初筛
```

## 局限与挑战

- **人工评测的内部一致性**：人类偏好也可能受 verbosity bias、sycophancy 影响，并非真正的"黄金标准"
- **LLM 裁判质量波动**：LLM-as-judge 的质量随裁判模型版本更新而变化，历史评测不可重现
- **专家 vs 普通用户**：专家评测与目标用户的偏好可能不一致，选择合适的评测者群体本身就是设计挑战
- **任务多样性**：没有一种评测方法能在所有任务上同时最优
- **资源约束下的选择**：学术研究者和工业界的资源差异导致评测深度不可比
