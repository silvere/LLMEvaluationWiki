# SelfCheckGPT: Zero-Resource Black-Box Hallucination Detection for Generative Large Language Models

> 提出基于采样一致性的零资源幻觉检测方法，无需外部知识库即可识别黑盒 LLM 输出中的非事实句子。

## 核心主张
- [CLAIM]: 若 LLM 对某概念有真实知识，多次采样的输出在事实上应高度一致；反之，幻觉内容会在不同采样间呈现不一致性。
- [CLAIM]: SelfCheckGPT 无需访问模型内部参数或外部知识库，仅通过多次采样即可实现零资源的幻觉检测，适用于所有黑盒 LLM。
- [CLAIM]: 在 GPT-3 生成的人物传记段落上测试，该方法在句子级非事实性识别和段落级事实性排序上均优于已有方法。

## 方法论摘要

SelfCheckGPT 的核心思路是：对同一输入多次采样 LLM 的输出，通过衡量不同采样之间的事实一致性来判断某一陈述是否为幻觉。具体而言，作者探索了多种一致性衡量方式，包括基于 BERTScore 的语义相似度比较、基于 n-gram 的文本重叠度，以及利用另一个 LLM 来判断待检测句子是否与其他采样结果矛盾。

实验数据集为 WikiBio，研究者让 GPT-3 生成维基百科人物传记，并与真实传记进行人工标注对比。评估分为两个层面：句子级别（判断每句话是否为非事实陈述）和段落级别（对段落整体事实性进行排序）。

该方法的关键优势在于其黑盒属性——不需要 logit 或概率输出，仅依赖文本采样，因此可直接应用于 GPT-3/4 等只提供 API 的商业模型。

## 数据与结果

- 测试数据集：WikiBio（GPT-3 生成传记 + 人工标注）
- 评估模型：GPT-3（text-davinci-002）
- 句子级任务：AUC-PR 优于基线方法（如 NLI-based 方法）
- 段落级任务：Spearman 相关系数优于 perplexity 和 length-based 基线
- 采样次数通常为 20 次，计算成本随采样数线性增长

## 局限与争议

- 多次采样带来额外的推理成本，对于高延迟或高费用场景不友好。
- 一致性并不等同于正确性：若模型在某错误信息上训练充分，可能会一致地产生相同的幻觉内容，从而绕过检测。
- 实验仅在传记领域（WikiBio）验证，对其他领域（如医学、法律）的泛化能力尚未充分评估。
- 检测粒度为句子级，对于句内部分幻觉（如数字、日期错误）的识别能力有限。
