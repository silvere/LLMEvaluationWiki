# G-Eval: NLG Evaluation using GPT-4 with Better Human Alignment

> 提出结合链式思维与填表范式的 GPT-4 自动评估框架 G-Eval，在文本摘要和对话生成任务上显著提升与人类评分的相关性。

## 核心主张
- [CLAIM]: 传统自动评估指标（BLEU、ROUGE 等）与人类判断的相关性较低，尤其在需要创造性和多样性的生成任务上表现不佳；GPT-4 驱动的 G-Eval 在摘要任务上 Spearman 相关系数达 0.514，大幅优于已有方法。
- [CLAIM]: G-Eval 采用"链式思维 + 填表打分"的双步范式：先让 LLM 生成详细的评估步骤，再按步骤对输出进行评分，从而比直接提示评分更稳定。
- [CLAIM]: 基于 LLM 的评估器存在明显的"自我偏好"偏差——倾向于给 LLM 生成的文本打更高分，即便该文本并不更优质，这是当前 LLM-as-Judge 范式的重要潜在风险。

## 方法论摘要

G-Eval 的工作流程分为两步：首先向 GPT-4 提供评估任务描述和待评估维度（如连贯性、相关性、流畅性），让模型以链式思维生成结构化的评估步骤；随后将这些步骤与待评估文本一并输入，要求模型按 1-5 分打分，并以 token 概率加权平均作为最终得分，以减少离散打分的方差。

实验在两个 NLG 任务上进行：文本摘要（SummEval 基准）和对话生成（TOPICAL-CHAT 数据集）。评估维度包括连贯性、一致性、流畅性和相关性等，以 Spearman 和 Kendall 相关系数衡量与人工评分的对齐程度。

论文还对比了 GPT-3.5 和 GPT-4 在该框架下的表现差异，并专门设计实验分析 LLM 评估器的自我偏好问题，发现 GPT-4 对 GPT-3.5 生成的文本评分显著高于人类偏好。

## 数据与结果

- 摘要任务：SummEval 基准，Spearman 相关系数 0.514（一致性维度），超越所有已有自动化方法
- 对话生成：TOPICAL-CHAT，多维度相关性均有提升
- 自我偏好实验：GPT-4 对 GPT-4 生成内容评分比人类高约 10-15 个百分点
- 使用 token 概率加权平均比整数打分降低约 50% 的得分方差

## 局限与争议

- 自我偏好偏差是核心风险：使用 GPT-4 评估 GPT-4 生成内容会产生系统性高估，影响评估公正性。
- 评估结果依赖具体的 prompt 设计，不同提示措辞可能带来显著得分差异，可重复性存疑。
- 实验集中于英文摘要和对话生成，跨语言和跨任务的泛化能力尚未充分验证。
- 调用 GPT-4 API 的成本较高，大规模评估时经济性不如传统指标。
