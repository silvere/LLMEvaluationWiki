# Likert 量表

> 用于测量态度或主观感受的有序评分量表，通常为 5 分或 7 分制，是 LLM 人工评测中收集质量判断的标准工具。

## 定义

Likert 量表（Likert Scale）由 Rensis Likert 于 1932 年提出，提供一组有序的态度选项，要求受访者对陈述语句选择同意程度。在 LLM 评测语境中，通常用于收集对模型输出质量的主观评分。

常见形式：
- **5分制**：1=非常差 / 2=较差 / 3=一般 / 4=较好 / 5=非常好
- **7分制**：1-7，提供更细粒度区分
- **双极量表**：中间值为中性，两端为对立极（如"非常不同意"到"非常同意"）

**Likert 量表 vs Likert 题目**：严格意义上，"Likert 量表"是多题合并计算总分的综合量表；单独一道评分题称为 Likert 式题目（Likert-type item）。NLP 评测中通常是后者。

## 重要性（在 LLM 评测中）

Likert 评分是 LLM 主观评测的核心工具，用于量化难以客观测量的维度：

1. **多维度质量评测**：流畅性（Fluency）、连贯性（Coherence）、相关性（Relevance）、有用性（Helpfulness）、安全性（Safety）各用独立 Likert 题目
2. **RLHF 数据收集**：偏好标注中，标注者对每个回答的质量打分，辅助成对比较
3. **人工评测标准化**：将模糊的"好/坏"判断量化为可聚合、可统计的数值
4. **LLM-as-judge 验证**：GPT-4 等裁判模型给出的评分通常也采用 Likert 格式

## 主要方法/实现

**评分维度设计**：LLM 评测常见维度（参考 MT-Bench 等协议）：
- 准确性（1-10）：回答是否正确/符合事实
- 有用性（1-10）：对用户问题的实际帮助程度
- 安全性（1-5）：是否含有害/偏见内容
- 流畅性（1-5）：语言是否自然流畅

**结果汇总**：多标注者打分通过算术平均或中位数汇总，并报告方差/标准差以反映一致性。

**Likert 数据分析注意事项**：
- 单题 Likert 应视为序数数据，不应直接计算均值（尽管实践中常见）
- 一致性测量应使用 Krippendorff's α（序数模式）而非百分比一致率
- 需报告分布直方图，均值可能掩盖双峰分布

## 局限与挑战

- **中间选项偏向**：标注者倾向于选择中间值，导致分数集中在 3/5 或 4/7，区分度不足
- **标定差异**：不同标注者的"严格程度"不同，有人的 4 等于另一人的 3，导致分数不可比
- **顺序效应**：量表端点标签措辞影响评分分布，"极差"与"非常差"作为最低分会产生不同效果
- **不捕获理由**：Likert 分数无法说明"为什么"打这个分，缺少解释性
- **语言文化差异**：不同文化背景标注者使用量表端点的习惯不同，影响跨组比较
