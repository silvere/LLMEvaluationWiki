# 智能体评测前沿

## 背景

AI 智能体（Agent）是指能够自主规划、使用工具、处理环境反馈并完成多步骤任务的 LLM 系统。随着 GPT-4o、Claude 3.5 Sonnet 等模型具备强大的工具调用能力，以及 LangGraph、AutoGen、CrewAI 等代理框架的普及，AI 代理从实验室进入了实际应用。然而，代理评测方法论相对于代理应用的发展严重滞后。

## 代理评测的核心挑战

**评测复杂性爆炸**：
传统评测是"输入 → 输出"的单步评测。代理任务是多步骤的：规划 → 工具调用 1 → 结果处理 → 工具调用 2 → ... → 最终答案。每一步都可能出错，且错误会累积传播。

**环境依赖性**：
代理任务需要真实的执行环境（浏览器、文件系统、数据库、API 等）。构建和维护这些环境成本高，且不同环境版本会影响结果。

**评测指标的两难**：
- 只看最终结果（任务是否完成）：忽略了中间步骤的效率，且成功/失败是二元的，缺乏细粒度
- 评测中间步骤：需要定义"正确的推理过程"，但代理往往存在多种有效路径

**随机性问题**：
代理任务涉及多次 LLM 采样，同一任务不同运行的成功率可能差异显著。评测需要多次运行取平均，计算成本极高。

## 主要评测框架现状

**SWE-bench / SWE-bench Verified**：
软件工程代理最权威的评测基准，要求模型解决真实 GitHub 仓库中的 Issue。SWE-bench Verified 是经人工验证的子集，减少了原版中有歧义任务的干扰。主要挑战：运行成本高，需要真实代码执行环境。

**WebArena / VisualWebArena**：
网页代理的标准评测，提供模拟的电商、论坛、Wiki 等网站环境。任务如"在购物网站搜索并购买指定商品"。成功率是二元的（完成/未完成），难以评测部分完成情况。

**AgentBench**：
覆盖 8 种代理任务环境（操作系统命令行、数据库查询、网页浏览、知识图谱、数字游戏等），提供综合代理能力评分。

**GAIA**：
要求模型结合网页搜索、文件处理、计算等多种工具完成真实世界问题。分三个难度级别，顶级模型在最难级别上的通过率仍然很低（<30%）。

**τ-bench（Tau-bench）**：
聚焦于真实客服和零售场景的代理任务，强调与用户交互和数据库操作的结合，是最接近商业代理应用场景的评测之一。

## 开放问题

**如何评测代理的"规划能力"**：
现有框架大多只评测最终结果，无法区分"靠运气完成"和"通过高质量规划完成"。

**长时间任务评测**：
一些真实世界任务需要数十步操作，评测成本极高，且很难区分是任务本身失败还是评测环境问题。

**代理可靠性评测**：
相同任务重复 10 次，代理的通过率可能从 30% 到 80% 不等。如何定义和评测代理的"可靠性"尚无共识。

**工具选择效率**：
代理可能用 20 步完成一个人类 3 步就能完成的任务。评测应该同时考虑完成质量和步骤效率。

## 展望

代理评测领域需要从"是否完成任务"向"如何完成任务"的更细粒度评测演进。随着代理应用从实验走向生产，面向真实工作场景（而非玩具任务）的评测基准将成为更重要的需求。
