# FairCoder

> 由中国科学技术大学等机构发布的代码生成社会偏见评测基准，通过函数生成与单元测试生成两类任务，系统测量 LLM 在涉及敏感人口属性时是否产生歧视性代码输出（arXiv:2501.05396，2025）。

## 概述

FairCoder（Evaluating Social Bias of LLMs in Code Generation）是专注于代码生成公平性评测的综合基准，于 2025 年 1 月发布于 arXiv。该工作针对一个被长期忽视的问题：当 LLM 生成涉及不同人口群体（性别、种族、年龄、社会经济地位等）的代码时，是否存在系统性偏见？

与通用文本偏见评测不同，FairCoder 聚焦于代码的功能性偏见——即模型所生成的函数逻辑、条件判断或测试用例是否对特定群体更为苛刻或不公平。研究团队设计了两类主要评测场景：函数实现（Function Generation）和单元测试生成（Test Case Generation），涵盖多种真实世界的应用场景（如招聘筛选、贷款审批、医疗诊断等）。

研究发现，即使是最先进的代码大模型，在涉及年龄、社会经济状况等较少被关注的敏感属性时也会暴露出显著偏见；而性别和种族等常见偏见则因模型已习得规避策略而在表面上有所减弱，但通过反事实测试仍可检测到。

## 任务格式

- **任务类型一：函数生成**——给定含有敏感人口属性的功能描述，要求模型生成函数实现，评估输出代码是否对不同群体存在差别对待
- **任务类型二：单元测试生成**——给定一段函数代码，要求模型为不同人口群体生成测试用例，检验测试覆盖率和测试严格度是否存在群体差异
- **敏感属性覆盖**：性别、种族、年龄、社会经济地位、收入水平等多个维度
- **评估方式**：自动化指标为主，结合人工抽查验证
- **测试模型**：Llama、Mistral、Qwen、GPT 系列等主流代码 LLM

## 主要指标

FairCoder 提出三项核心公平性度量指标：

- **拒绝率（Refusal Rate）**：模型拒绝生成含敏感属性代码的频率，衡量模型对公平性的自我约束程度
- **偏好熵（Preference Entropy）**：衡量模型在不同人口群体之间的输出多样性，熵值越低说明模型越倾向于特定群体
- **反事实差异（Counterfactual Difference）**：将输入中的人口属性替换为不同群体后，测量模型输出的变化幅度，直接量化歧视性输出

**主要发现**：
- 单元测试生成任务中偏见问题比函数生成更为严重
- 模型对年龄和社会经济地位的偏见大于对性别和种族的偏见
- 所有测试模型均表现出不同程度的社会偏见，无一例外

## 局限性

- **场景覆盖有限**：目前评测场景主要聚焦于几类常见应用（招聘、金融、医疗），难以穷举所有可能产生偏见的代码生成场景
- **英语为主**：评测提示以英语为主，对多语言代码生成偏见的覆盖不足
- **偏见定义存在主观性**：何为"公平"的代码输出在不同文化和法律语境下可能存在分歧
- **静态评测局限**：代码生成偏见随模型版本迭代可能发生变化，静态基准需定期更新
- **对抗性提示未覆盖**：未系统测试通过越狱或间接提示引发偏见的场景
