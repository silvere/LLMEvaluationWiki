# 偏好评测（Preference Evaluation）

> 通过收集人类或 LLM 对模型回答的成对偏好判断来评测 LLM 质量的方法，是 Chatbot Arena 等排行榜和 RLHF 数据收集的核心范式。

## 定义

偏好评测（Preference Evaluation）将质量评估问题转化为相对比较问题：给定同一输入，展示两个（或多个）不同模型的回答，要求评判者选择更好的那个，或标注偏好强度。

与绝对评分的区别：
- **Likert 评分**：对每个回答独立打分（绝对判断）
- **偏好评测**：在回答对之间选择更好的（相对判断）

相对判断的优势：排除了标注者之间的绝对标准差异（"标定偏移"），更稳定可靠。

偏好数据的主要形式：
- **二元偏好**：A 更好 / B 更好 / 平局
- **多级偏好**：A 明显更好 / A 稍微更好 / 相当 / B 稍微更好 / B 明显更好
- **多维度偏好**：分别在有用性、安全性、流畅性等维度评判

## 重要性（在 LLM 评测中）

偏好评测是当前 LLM 评测最重要的方法论之一：

1. **RLHF 的数据基础**：人类偏好标注直接驱动奖励模型训练，偏好评测质量决定 RLHF 效果
2. **Chatbot Arena 的理论基础**：用户真实偏好投票（而非专家评分）被认为是最贴近实际价值的排行榜
3. **胜率指标的来源**：所有胜率、ELO 评分最终都来源于成对偏好判断
4. **主观任务的唯一可行评测**：创意写作、对话风格等任务无客观标准，偏好是唯一可量化的质量信号

## 主要方法/实现

**人类偏好评测标准流程**：
1. 确定评判维度（有用性/无害性/诚实性）
2. 为每个维度提供明确评判标准
3. 随机分配成对比较给标注者
4. 收集足够多的比较（推荐每模型对 ≥ 500 次比较）
5. 用 ELO/Bradley-Terry 模型将成对结果转换为全局排名

**LLM-as-Judge 偏好评测**（AlpacaEval、MT-Bench）：
```
Prompt: 给定以下问题，评判哪个助手的回答更好。
问题：{question}
助手A：{response_A}
助手B：{response_B}
请解释你的判断理由，然后给出最终裁决（A/B/平局）。
```

**Bradley-Terry 模型**：从成对偏好数据估计每个模型的"能力参数"，转化为可比较的评分。

## 局限与挑战

- **评判者偏差**：位置偏差、冗长偏差等系统性偏差影响偏好判断的可靠性
- **偏好的不传递性**：A>B, B>C 不一定意味着 A>C，循环偏好使排名不稳定
- **任务分布依赖**：同一对模型在不同任务分布下偏好结果可能相反
- **表面质量 vs 深层价值**：偏好可能被表面特征（流畅性、长度）主导，而非内容价值
- **对齐税的不可见性**：偏好可能更高但事实准确性更低，两者难以同时优化
