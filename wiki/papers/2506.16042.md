# OSWorld-Human: Benchmarking the Efficiency of Computer-Use Agents

> UCSD WukLab 2025-06 推出的 **[[OSWorld]] 效率评测扩展**。**手工标注每个任务的 human-determined trajectory（最少步数）**，揭示「即使最强 agent 也需要 2.7-4.3x 人类必要步数」+ 「end-to-end 延迟数十分钟」的可用性危机。

## 论文核心贡献

1. **OSWorld 人工标注**：每个 OSWorld 任务标注人类最少完成步数
2. **首个 efficiency benchmark**：从「能不能完成」转向「多浪费」
3. **16 个 agent 测评**：2.7-4.3x step overhead
4. **Latency 拆解**：planning / reflection / judge 三类 LLM 调用占主要延迟
5. **3x 后期 step 延时膨胀**：agent 步数越长每步越慢（context 累积成本）

## 评测圈意义

- 是 [[OSWorld]] 上「pass rate」之外**第一个效率维度**
- 引入「人类基线 = 最少步数」是 agent 评测的范式贡献
- 与 [[AgentRewardBench]] 都是 2025 年「agent 评测的二阶段」（不只评成败，评质量）
