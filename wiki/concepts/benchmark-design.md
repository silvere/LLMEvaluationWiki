# 基准设计原则（Benchmark Design）

> 构建有效 LLM 评测基准的方法论原则，涵盖任务选择、难度校准、防污染机制、可复现性保障和长期维护等关键设计考量。

## 定义

评测基准（Benchmark）是一组标准化的任务、数据集和评测协议的集合，用于系统比较不同 LLM 的能力。基准设计需在多个维度做出权衡：覆盖面 vs 深度、难度 vs 可解释性、防污染 vs 可公开使用、自动化 vs 有效性。

好的基准应满足：
- **效度（Validity）**：确实测量了目标能力
- **信度（Reliability）**：结果可复现，不同时间评测同一模型得到相近分数
- **区分度（Discriminability）**：能有效区分不同能力水平的模型
- **可解释性**：分数与实际能力有清晰对应关系

## 重要性（在 LLM 评测中）

基准是 LLM 研究进步的度量工具和导航系统，其设计质量直接影响：

1. **研究方向引导**：研究者趋向于优化基准得分，基准设计偏差会扭曲整个领域的研究方向
2. **工业部署决策**：企业依据基准选择模型，基准不反映实际需求则选择失当
3. **进步衡量**：若基准设计不当（如过早饱和），无法反映真实的技术进步
4. **污染风险管理**：公开基准必须内置防污染机制，否则排行榜迅速失效

## 主要方法/实现

**任务选择原则**：
- 覆盖多种能力维度（推理/知识/代码/对话/安全）
- 包含分层难度（易/中/难）以区分不同水平模型
- 任务来自真实使用场景而非人造场景

**难度校准方法**：
- 在人类（专家/非专家）上测试，确保任务对目标群体有适当难度
- 避免"数据集偏差"——某些任务可以通过浅层特征捕获而非真正理解

**防污染设计**：
- 保持私有测试集（公开训练集）
- 周期性更新题目（[[dynamic-benchmarks-concept]]）
- 发布时进行训练集污染扫描

**可复现性保障**：
- 明确 prompt 模板、示例数量、输出解析方式
- 公开评测代码（sacrebleu/lm-evaluation-harness 等标准框架）
- 设定种子确保随机性可控

**基准生命周期管理**：GLUE → SuperGLUE → BIG-Bench → MMLU 的演进展示了基准饱和-替换的周期，新基准设计应估计饱和时间线。

## 局限与挑战

- **完美防污染 vs 开放性矛盾**：完全私有化的基准失去社区参与价值，完全公开则污染风险高
- **单一分数的欺骗性**：聚合分数掩盖能力不均衡（在某些子任务极强、某些极弱）
- **任务选择的主观性**：任务选择反映设计者的能力观，不同社区对"重要能力"有不同定义
- **时间成本**：严格的基准构建（数据收集、人工验证、协议设计）需要大量资源
- **Goodhart 定律**：一旦基准广泛使用，它就会成为优化目标而失去度量价值
