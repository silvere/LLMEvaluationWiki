# GuardAgent: Safeguard LLM Agents by a Guard Agent via Knowledge-Enabled Reasoning

> 提出以"守卫代理"动态监控目标 LLM 代理行为的安全框架 GuardAgent，并引入医疗和网页场景下的两个新评测基准，在两个基准上分别达到 98% 和 83% 的防护准确率。

## 核心主张
- [CLAIM]: LLM 代理的行为安全无法仅靠静态规则约束，需要一个能够动态解析安全约束并将其转化为可执行防护代码的"守卫代理"来实现实时监控。
- [CLAIM]: GuardAgent 通过任务规划（将安全约束转化为执行计划）和代码生成（将计划映射为可运行的防护代码）两步流程，结合记忆模块存储历史经验，实现对目标代理行为的确定性安全检查。
- [CLAIM]: 在 EICU-AC（医疗代理访问控制）基准上防护准确率超过 98%，在 Mind2Web-SC（网页代理安全策略）基准上达到 83%，优于现有安全方法。

## 方法论摘要

GuardAgent 的设计基于"代理守卫代理"的元架构：在目标 LLM 代理执行任务的过程中，GuardAgent 实时监控每一步行为，判断其是否违反预设的安全约束（guard request）。安全约束可以是自然语言描述的访问控制规则、数据隐私要求或行为限制。

GuardAgent 的核心工作流程分为两步：首先分析安全约束，利用 LLM 将其分解为结构化的任务计划；然后将任务计划转换为 Python 防护代码，通过代码执行实现确定性的行为检查，避免纯 LLM 推理的随机性。为提升少样本情境下的推理能力，GuardAgent 维护一个记忆模块，存储来自历史任务的示例，并在推理时检索相关经验作为上下文示例。

论文同时贡献了两个新评测基准：EICU-AC 基于电子病历系统构建，评测医疗代理的数据访问控制；Mind2Web-SC 基于网页操作数据集构建，评测网页代理在执行用户指令时的安全策略遵守情况。

## 数据与结果

- 新基准数量：2 个（EICU-AC 和 Mind2Web-SC）
- EICU-AC 防护准确率：超过 98%
- Mind2Web-SC 防护准确率：83%
- 对比方法：优于直接提示 LLM 判断和已有安全过滤方法
- 代码执行确定性：通过代码执行而非 LLM 输出实现安全判断，减少随机性

## 局限与争议

- 代码生成步骤本身可能产生错误，导致防护代码执行失败或误报，83% 的准确率反映了这一局限。
- 安全约束的表达需要一定的规范性，模糊或复杂的自然语言约束可能导致任务规划出错。
- 当前测试场景（医疗、网页）相对特定，对其他高风险代理场景（金融、法律）的泛化性未经验证。
