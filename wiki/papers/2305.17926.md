# Large Language Models are not Fair Evaluators

> 北大 + 腾讯云 AI 2023-05 经典论文。**系统性揭示 LLM-as-Judge 的位置偏差**：仅仅交换候选答案的出现顺序，就能让 GPT-4 的偏好结论反转。提出 Multiple Evidence Calibration + 多轮 swap 聚合 + 人工 review 三层校正。

## 论文核心贡献

1. **首次系统证明 LLM judge 的 position bias**：response order 决定排名
2. **MEC（Multiple Evidence Calibration）方法**：先让 judge 生成多份 evidence 再打分
3. **Cross-ordering aggregation**：交换顺序跑多次取均值
4. **在 Vicuna-Benchmark 上验证**：校正后与人类一致性提升

## 影响

- 是 [[LLMBar]] / [[JudgeBench]] / [[Prometheus]] 等后续 judge 评测的奠基论文
- 业界 LLM-as-Judge 标配「至少跑 2 次 swap」即源自此文
- [[RewardBench]] / [[Arena-Hard]] 的位置随机化均借鉴此思路

## 限制 / pitfall

- MEC 计算成本是裸 judge 的 2-4x
- 对 verbosity / self-preference 等其他偏差未覆盖（[[LLMBar]] 续作）
