# Program Synthesis with Large Language Models

> Google Brain 提出 MBPP（Mostly Basic Python Problems）基准，包含 974 道基础 Python 编程题，并系统评估大语言模型的程序合成能力。

## 核心主张
- [CLAIM]: 论文提出 MBPP 基准，包含 974 道由众包工作者编写的 Python 编程题，每题含问题描述、参考解答和三个测试用例。
- [CLAIM]: 题目整体难度低于 APPS 和竞技编程题，定位于"大多数初级程序员能解决"的基础题目，适合衡量模型的基础编程能力。
- [CLAIM]: 使用 few-shot 提示（展示少量示例题目-解答对），GPT 系列模型在 MBPP 上表现优于零样本，说明示例对代码生成有帮助。
- [CLAIM]: 论文还探讨了用测试用例验证生成代码的方法，以及通过多次采样提升通过率的策略。

## 方法论摘要

题目由 Amazon Mechanical Turk 众包工作者编写，经人工审核筛选。标准划分：训练集 374 题、验证集 90 题、测试集 500 题（另有 10 道调试题）。评测使用 pass@k，代码在沙箱中运行测试用例，全部通过计为正确。

## 数据与结果

- 总题数：974（测试集 500）
- 平均每题测试用例数：3 个
- 评测模型包含 GPT-3（175B）和更小规模模型
- GPT-3（few-shot）pass@1：约 47%（具体数字依论文实验设置略有不同）
- 相比 HumanEval，MBPP 题目更简短，平均难度较低

## 局限与争议

- 每题仅 3 个测试用例，测试覆盖率偏低，通过测试不代表完全正确。
- 众包来源导致题目质量存在一定不一致性。
- 部分题目描述不够严谨，存在歧义。
- 作为相对简单的基准，随着模型能力提升，MBPP 已接近饱和。
