# SUPER

> 由 Allen Institute for AI 发布的 agent 评测基准，评测 LLM agent 能否自主完成从真实研究代码库中拉取、配置并执行任务的全流程。

## 概述

SUPER（Setting Up and Executing tasks from research rePosItories）由 Allen Institute for AI（AI2）于 2024 年发布，收录于 EMNLP 2024 主会议论文集。它是第一个专门评测 LLM agent 在真实机器学习与自然语言处理研究仓库中"端到端完成任务"能力的基准。

该基准的动机来自实际科研痛点：研究者使用同行代码时，往往需要先配置环境、理解数据格式、修改训练脚本，才能运行复现实验。SUPER 将这一过程形式化为可自动评测的任务，覆盖依赖安装、配置修改、脚本调试、模型训练触发到结果提取等多个子步骤，能有效区分"能理解代码"与"能实际跑通代码"这两层能力。

基准数据来自 GitHub 上的真实 ML/NLP 研究仓库，由专家标注者撰写任务描述并给出参考解法。三类问题集分别支持不同规模的评测：专家集提供最高质量参考解，子问题集聚焦特定操作难点，自动生成集则提供大规模开发用例。评测结果显示，GPT-4o 在端到端集上仅解决 16.3% 的问题，说明该任务对当前最优模型仍具有显著挑战性。

## 任务格式

- **题目总量**：45 道端到端专家问题 + 152 道子问题 + 602 道自动生成问题
- **题目类型**：自由形式的 agent 任务（任务描述 → 自主规划 → 执行操作 → 验证结果）
- **评估方式**：程序化验证（执行结果是否符合参考输出，如模型精度、文件内容等）
- **子问题类型**：环境配置、数据处理、训练脚本修改、超参调整、结果提取等

## 主要指标

- **端到端成功率**：agent 独立完成完整任务的比例（GPT-4o：16.3%）
- **场景成功率（Scenario）**：部分子步骤完成的比例（GPT-4o：46.1%）

## 局限性

- 任务主要覆盖 ML/NLP 领域仓库，对生物信息、物理模拟等其他科学计算场景的迁移能力有待验证
- 端到端评测执行代价高，不适合在受限计算环境中大规模自动测试
- 自动生成的 602 个问题质量参差不齐，部分可能存在歧义或可行性问题
