# MLGym-Bench

> Meta 发布的机器学习科研 Agent 评测框架与基准，包含 13 个真实 ML 科研任务，评测 AI Agent 进行模型训练、实验设计、超参搜索等全流程科研工作的能力。

## 概述

MLGym-Bench 于 2025 年 2 月发布，是 Meta FAIR 研究团队推出的首个将机器学习（ML）科研任务形式化为 Gym 环境的评测框架与基准。其目标不仅是评测现有前沿 LLM 在 ML 科研任务上的能力，还为训练更强的科研 Agent 提供可扩展的强化学习环境。

传统 AI Agent 评测多集中于信息检索、工具调用或代码生成等单步或短链任务，而 ML 科研是一种典型的长时域、高复杂度开放性任务——研究者需要提出假设、搜索数据、实现算法、训练模型、分析结果，并在多轮迭代中优化方案。MLGym-Bench 正是针对这一完整科研工作流设计的，要求 Agent 在给定研究目标和初始代码/数据后，通过自主探索实现性能提升。

MLGym-Bench 包含 13 个多样化的开放式 ML 科研任务，跨越计算机视觉（图像分类、目标检测）、自然语言处理（文本分类、语言模型训练）、强化学习（Atari、CartPole 等经典环境）、博弈论（纳什均衡求解）等多个 AI 子领域。每个任务都提供：任务描述与评测指标、初始 baseline 代码、可用计算资源约束，以及用于自动评分的奖励函数。

对 Claude-3.5-Sonnet、Llama-3.1 405B、GPT-4o、o1-preview、Gemini-1.5 Pro 等前沿模型的评测结果揭示：当前模型确实能够改进给定的 baseline（主要通过找到更优的超参数组合），但几乎无法提出真正的新假设、设计新颖算法架构或实现实质性的方法创新。这一结论深刻揭示了当前 AI 系统在创造性科研能力上的根本局限。

## 任务格式

- **任务总量**：13 个多样化开放式 ML 科研任务
- **任务类别**：计算机视觉、NLP、强化学习、博弈论等多个 AI 子领域
- **输入内容**：任务描述 + 初始 baseline 代码 + 数据集访问权 + 计算资源约束
- **评估方式**：每个任务有独立的评测指标（如分类准确率、游戏分数等），与 baseline 的相对提升作为核心度量
- **交互方式**：Agent 可执行代码、修改文件、调用工具，在 Gym 环境中多轮迭代
- **时间限制**：每个任务设置最大交互步数或时间限制

## 主要指标

- **Baseline 相对提升率**：Agent 最终提交结果相对于初始 baseline 的性能提升百分比，核心排名指标
- **任务成功率**：在所有 13 个任务上超越 baseline 的任务数比例
- **方法新颖性**：定性分析 Agent 是否提出了超越超参搜索的真正方法创新（当前模型几乎为零）

## 局限性

- 13 个任务覆盖范围有限，部分 ML 子领域（如生成模型、多模态学习）尚未涵盖
- 评测结果高度依赖计算资源分配，在不同算力条件下结果可能不具可比性
- "方法新颖性"难以自动量化，需要人工判断，影响可重复性
