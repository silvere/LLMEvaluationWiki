# 动态基准（Dynamic Benchmarks）

> 通过定期更新、替换或动态生成测试题目来防止训练集污染和评测过拟合的基准设计范式，是对抗静态基准快速失效的根本解决方案。

## 定义

动态基准（Dynamic Benchmarks）与传统静态基准的核心区别在于：测试集不是固定不变的，而是通过某种机制持续更新，使模型无法通过记忆固定题目获得高分。

主要实现范式：
1. **定期人工更新**：由人工定期收集新题目并替换旧题目（如 LiveBench 每月更新）
2. **自动生成**：用算法或 LLM 自动生成语义等价但表面不同的新题目
3. **时间截点依赖**：基于最新事件的评测，知识截止后的内容天然是"新鲜"的（如 FreshQA）
4. **用户驱动更新**：Chatbot Arena 模式，用户提交真实问题，问题集持续增长

## 重要性（在 LLM 评测中）

动态基准是 LLM 评测生态可持续性的关键：

1. **对抗污染的根本方案**：静态基准一旦公开即面临污染风险，动态更新使持续跟踪评测集变得不经济
2. **反映能力进步**：不断引入更难的新题目，基准不会饱和，能持续区分模型能力差异
3. **追踪真实世界对齐**：基于最新知识的评测更贴近用户真实需求，而非历史题库
4. **减少评测过拟合激励**：若题目会变化，专门针对当前基准进行微调的性价比大幅下降

## 主要方法/实现

**LiveBench（White et al., 2024）**：
- 每月从最新 arxiv 论文、新闻等提取新题目
- 问题基于最新发布的内容，天然无法被预训练数据污染
- 涵盖推理、数学、编程、指令跟随等多个类别

**FreshQA**：
- 专注于最新事件的问答，分为"快速变化"和"缓慢变化"两类问题
- 随时间更新正确答案，测试模型对时事的掌握

**DynaEval / Dynabench**：
- 人机协作动态构建：人类构造对当前最优模型失败的样本
- 随着模型进步，评测难度自适应提升

**题目轮换策略**：
```
# 每个评测周期：
1. 保留 70% 历史题目（维持历史可比性）
2. 新增 30% 新题目（引入新鲜内容）
3. 剔除所有模型准确率 >95% 的饱和题目
```

## 局限与挑战

- **历史可比性难题**：题目变化后，不同时期的分数无法直接比较，难以追踪进步轨迹
- **构建成本高**：高质量题目的持续生成需要持续的人力或可靠的自动化流程
- **质量一致性**：不同批次题目的难度和质量难以保持一致，影响评测公平性
- **自动生成质量**：LLM 自动生成的题目可能存在错误、歧义或难度不均
- **短期性能焦虑**：动态基准下模型分数会波动，不适合作为短期迭代的即时反馈
