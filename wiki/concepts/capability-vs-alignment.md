# 能力与对齐的权衡（Capability vs Alignment）

> 更强的 LLM 能力是否必然带来更高安全风险的核心争议，以及如何在评测中同时衡量两个维度、理解两者之间的实证关系。

## 定义

能力（Capability）与对齐（Alignment）的权衡讨论来自 AI 安全领域的核心担忧：

- **对齐税（Alignment Tax）假说**：通过 RLHF、宪法 AI 等对齐训练提升安全性，会以降低模型能力（如推理准确率、有用性）为代价
- **能力-危险性正相关担忧**：模型越强大，潜在的错误使用（恶意行为者利用）造成的危害越大
- **对齐随能力扩展的乐观假说**：更聪明的模型可能更容易理解并遵守人类价值观

实证研究结果复杂：InstructGPT 等早期对齐模型确实在某些能力基准上略低于基础模型；但更近的研究（如 Llama 2 Chat vs Llama 2 Base）发现对齐后能力基准上有所提升。

## 重要性（在 LLM 评测中）

这一权衡直接影响评测设计的基本理念：

1. **双目标评测框架**：评测必须同时报告能力（MMLU、HumanEval 等）和对齐（TruthfulQA、安全红队结果）
2. **权衡曲线可视化**：研究能力-安全的 Pareto 前沿，识别能力相当但更安全（或安全相当但更强）的模型
3. **对齐税测量**：量化从基础模型到对齐模型的能力退步幅度，评估对齐效率
4. **危险能力评测**：发布前的安全评估需要积极寻找可能被滥用的危险能力（如 CBRN 援助、网络攻击）

## 主要方法/实现

**评测框架设计**：
```
能力评测：
- 推理：GSM8K, MATH, ARC-Challenge
- 知识：MMLU, TriviaQA
- 代码：HumanEval, MBPP

对齐/安全评测：
- 无害性：HarmBench, ToxiGen
- 诚实性：TruthfulQA, FactScore
- 谄媚：Sycophancy 测试集

综合权衡分析：
- 绘制能力分数 vs 安全分数的散点图
- 计算对齐后各能力指标的相对变化 (%)
```

**对齐税测量**：
```
alignment_tax = (base_capability - aligned_capability) / base_capability
```

**危险能力评测**（Anthropic 的方法论）：在部署前针对性测试：
- 大规模杀伤性武器辅助能力
- 网络攻击能力
- 自我复制和逃脱能力
- 欺骗能力

## 局限与挑战

- **能力定义的多维性**："能力"包含推理、知识、代码等多个维度，对齐对各维度影响不均
- **对齐评测的滞后性**：安全威胁是动态演进的，今日的对齐评测可能无法覆盖未来的风险场景
- **商业压力与安全压力的冲突**：对齐降低有用性会影响商业竞争力，形成减少对齐的激励
- **危险能力检测的不完整性**：无法穷举所有危险用途，评测只能发现已知风险
- **文化与价值观多元性**：不同文化对"对齐"目标的定义不同，难以构建普遍适用的评测标准
