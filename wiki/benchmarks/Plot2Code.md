# Plot2Code

> 评测多模态大模型从图表图像生成可执行可视化代码能力的基准，测试视觉理解与代码生成的协同推理。

## 概述

Plot2Code 于 2024 年提出，是评测多模态大模型**图表代码生成**能力的专项基准。该基准的任务是：给定一张数据可视化图表（如 matplotlib/seaborn 生成的折线图、柱状图、散点图等），要求模型生成能够复现该图表的可执行 Python 代码。

这项任务要求模型同时具备强大的视觉感知能力（准确理解图表的视觉属性：颜色、布局、图例、数值标注等）和代码生成能力（生成正确的数据可视化代码），是视觉-代码协同推理的典型应用场景。

Plot2Code 覆盖了多种常见图表类型（折线图、柱状图、饼图、散点图、热图等）和多种可视化库（matplotlib、seaborn），并采用代码执行后的图像相似度（SSIM、pixel-level comparison）评测生成代码的质量，而非仅依赖代码文本匹配。

评测结果显示，即使是 GPT-4V 在该任务上的代码可执行率和图像相似度得分也远低于人类，尤其在复杂图表（多子图、自定义样式）上差距显著。

## 任务格式

- **图表数量**：约 132 张参考图表（覆盖主要可视化类型）
- **图表类型**：折线图、柱状图、散点图、饼图、热图、多子图等 6 类以上
- **可视化库**：matplotlib（主要）、seaborn
- **输出格式**：可执行 Python 代码
- **评分方式**：
  - 代码可执行率（能否无错运行）
  - 图像结构相似度（SSIM）
  - 文字内容准确率（图表标签/标题是否正确）
  - GPT-4V 辅助综合评分

## 主要指标

- **代码可执行率（Pass Rate）**：生成代码能够无错执行的比例
- **图像相似度（SSIM）**：生成图像与参考图像的结构相似度
- **文字准确率**：图表中文字内容（标题、标签、数值）的匹配准确率
- **综合质量分**：GPT-4V 给出的综合视觉匹配分数

## 局限性

- 图表数量较少（约 132 张），统计代表性有限
- 评测依赖 Python 环境执行代码，不同版本库的兼容性可能导致评分偏差
- 侧重 matplotlib 生成图表，对其他可视化工具（Plotly、ECharts 等）覆盖不足
