# MuSR

> 多步骤软推理（Multistep Soft Reasoning）基准，通过算法生成包含长篇叙事的推理谜题，要求模型在复杂语境中追踪多个实体并进行链式推理。

## 概述

MuSR（Multistep Soft Reasoning）由 Sprague 等人于 2023 年发布，采用算法生成（algorithmically generated）方式构建评测数据，确保题目不会出现在模型训练数据中，从根本上规避数据污染问题。数据集包含三类推理场景：**谋杀悬疑推理**（murder mystery，给定案情描述，判断嫌疑人）、**团队合作任务分配**（team allocation，根据人员能力和约束条件分配任务）、**物体空间追踪**（object placement，追踪多步骤操作后物体的最终位置）。

MuSR 的设计核心是"软推理"（soft reasoning）——与形式化逻辑推理不同，MuSR 的推理过程嵌入在自然语言叙事中，线索以隐式、非结构化方式散布在长篇文本里（每道题的上下文长度约 1,000-3,000 词），模型需要从叙事中提取相关信息、维护实体状态、执行多步推理，最终给出选择题答案。

该基准揭示了 CoT（链式思维）提示对 LLM 软推理的重要性，同时也发现现有模型在需要长距离信息整合的多步推理任务上仍存在显著短板。MuSR 被收录在 Open LLM Leaderboard v2 的评测套件中，成为衡量模型推理能力的重要指标之一。

## 任务格式

- 格式：多选题（来自长篇叙事语境的选择题）
- 数据规模：约 756 道题（三类场景各约 250 道）
- 上下文长度：约 1,000–3,000 词/题
- 推理场景：谋杀悬疑 / 团队任务分配 / 物体空间追踪

## 主要指标

- **准确率（Accuracy）**：整体及各场景分别报告
- 随机基线约为 33%（三选一）或 50%（二选一），视具体任务而定

## 局限性

- **算法生成的自然度限制**：程序生成的叙事文本可能显得刻板或不自然，与真实人类叙事存在风格差距
- **任务覆盖场景有限**：三类推理场景各有特色，但覆盖面较窄，不能代表所有软推理类型
- **长上下文依赖**：性能受模型有效上下文处理能力影响，难以区分推理能力与上下文处理能力的贡献
