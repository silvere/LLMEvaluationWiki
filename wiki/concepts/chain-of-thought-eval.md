# 思维链评测（Chain-of-Thought Evaluation）

> 评测 LLM 推理过程质量而非仅看最终答案的评测方法体系，关注推理步骤的正确性、连贯性和忠实性，是提升评测深度的重要方向。

## 定义

思维链（Chain-of-Thought，CoT）评测认为：仅评测最终答案是不够的，一个好的推理模型应该不仅能给出正确答案，还应该通过可解释的推理步骤得出答案。CoT 评测涵盖两个方向：

1. **CoT 作为能力激发工具的评测**：通过触发 CoT 来测量模型在推理任务上的性能上限（Wei et al., 2022）
2. **推理过程本身的质量评测**：评测思维链是否正确、是否忠实地反映了模型的"真实推理"

**推理过程评测维度**：
- **步骤正确性**：每个推理步骤是否逻辑有效
- **步骤充分性**：是否包含得出结论所需的所有关键步骤
- **忠实性（Faithfulness）**：推理链是否真实反映了模型产生答案的过程（还是事后合理化）
- **可验证性**：步骤是否可以被外部工具或人类验证

## 重要性（在 LLM 评测中）

CoT 评测标志着 LLM 评测从"是否能给出正确答案"向"是否能正确推理"的范式转变：

1. **可解释性评测**：在高风险决策场景中，推理过程的可解释性与最终答案同等重要
2. **过程奖励模型（PRM）**：OpenAI 的 PRM800K 和 DeepMind 的研究表明，训练过程奖励模型比结果奖励模型更能提升推理能力，推动了推理步骤评测的需求
3. **幻觉溯源**：错误的中间推理步骤是最终幻觉答案的可追溯来源
4. **评测鲁棒性**：若模型通过错误推理得出正确答案（运气正确），纯结果评测会产生误导

## 主要方法/实现

**结果导向 CoT 评测**（传统方式）：
```python
# 仅比较最终答案
prompt = "解题：{question}\n让我们一步步想。"
response = model.generate(prompt)
final_answer = extract_answer(response)  # 从 CoT 中提取答案
correct = (final_answer == gold_answer)
```

**过程奖励模型（PRM）评测**：
- 在每个推理步骤后标注"该步骤正确/错误"
- 训练 PRM 预测每步的正确性
- 比较 PRM 与人工步骤标注的一致性

**忠实性评测**（Turpin et al., 2023 发现）：
- 在 CoT 中插入误导性信息，观察最终答案是否被影响
- 若答案随 CoT 内容变化，说明 CoT 反映了推理过程（高忠实性）
- 若答案不变，说明 CoT 是事后合理化

**自动 CoT 质量评测**：
- 用 LLM 裁判逐步评分推理链
- 检测逻辑跳跃、循环论证、无关步骤等缺陷
- 计算步骤级别的 F1（实际步骤与参考步骤的匹配）

## 局限与挑战

- **CoT 忠实性的根本问题**：模型的实际"计算过程"在 Transformer 内部，生成的文字推理可能并不反映真实推理（"事后叙事"问题）
- **步骤粒度标准不一**：不同研究对"一步"的粒度定义不同，影响步骤级评测的可比性
- **参考推理链的稀缺性**：过程奖励模型需要大量步骤级标注数据，成本极高
- **复杂推理的验证困难**：对于专业领域（如高等数学证明），评测者本身可能无法验证中间步骤
- **CoT 诱导的幻觉**：长推理链增加了在中间步骤产生幻觉并导致最终答案出错的机会
