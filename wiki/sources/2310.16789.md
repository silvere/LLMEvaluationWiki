# Detecting Pretraining Data from Large Language Models

> 提出 Min-K% Prob 方法，通过计算文本中概率最低的 k% token 的平均对数概率来检测该文本是否曾出现在模型预训练数据中，无需访问训练数据或梯度。

## 核心主张
- [CLAIM]: Min-K% Prob 方法基于观察：已见过的文本通常没有极低概率的 token，而未见过的文本（或噪声插入版本）会包含低概率异常 token；通过仅分析概率最低的 k% token 的平均分，可以有效区分两类文本。
- [CLAIM]: 在多个数据集上的成员推断任务上，Min-K% Prob 优于直接使用平均 token 概率（Likelihood）和困惑度（Perplexity）等基线方法。
- [CLAIM]: 实验发现，WikiMIA 数据集中，GPT-NeoX、LLaMA 等开源模型对 Wikipedia 2023 年文章（发布时间晚于训练截止日）的检测准确率为 AUC 约 0.6-0.7，说明该方法有一定区分力但并不完美。
- [CLAIM]: Min-K% 方法不依赖访问训练数据或模型梯度，仅需模型推断接口，实用性较高。

## 方法论摘要

在 WikiMIA 基准（维基百科不同时间段文章，已知训练/测试划分）和 BookMIA 上评测。AUC（ROC 曲线下面积）作为成员推断检测性能指标。对比方法包括：Likelihood、Perplexity、zlib 压缩比等。

## 数据与结果

- 评测数据集：WikiMIA、BookMIA
- Min-K%（k=20）vs. Perplexity 基线：AUC 提升约 3-8%（依模型和数据集）
- LLaMA 在 WikiMIA 上：Min-K% AUC 约 0.68（Perplexity AUC 约 0.62）
- 整体 AUC 约 0.6-0.7，表明检测仍不够可靠，但优于随机

## 局限与争议

- AUC 0.6-0.7 仍接近随机，在实际使用中误报率较高。
- 该方法假设模型推断接口可用，对 API-only 模型有时会受 token 概率访问限制。
- 不同超参数（k 值选择）对不同模型效果差异较大。
- 训练数据污染检测是研究活跃领域，Min-K% 被后续方法进一步改进。
