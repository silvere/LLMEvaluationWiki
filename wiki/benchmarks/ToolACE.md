# ToolACE

> 面向工具调用精准性的 LLM 评测基准，覆盖工具选择、参数生成和多轮调用链的细粒度评测。

## 概述

ToolACE 是 2024 年提出的工具调用精准性评测基准，专注于从**精确性**角度评测 LLM 的工具调用能力。与关注端到端任务完成的代理基准不同，ToolACE 深入分解工具调用过程的各个环节，对每个步骤进行独立的精细化评测：工具选择是否正确、参数名称是否准确、参数值是否符合类型约束、参数值语义是否正确。

ToolACE 包含超过 26,507 条多样化的工具调用对话数据，涵盖 16,464 个不同的 API 工具，覆盖数十个工具类别（天气、金融、搜索、计算、地理等）。数据集通过自动化的工具模拟器和 LLM 辅助生成流程构建，同时配套了严格的质量过滤机制，确保数据质量。

ToolACE 同时作为训练数据集和评测基准发布，其配套的 ToolACE 模型（基于 LLaMA-3.1 微调）在 BFCL（Berkeley Function Calling Leaderboard）上展示了超越 GPT-4o 的工具调用精准性，证明了专项训练对工具调用能力的显著提升效果。

## 任务格式

- **输入**：用户自然语言请求 + 可用工具集（含函数签名和参数说明）
- **输出**：精确的工具调用（工具名 + 结构化参数 JSON）
- **规模**：26,507 条多轮对话，16,464 个不同 API 工具
- **调用类型**：单次调用、并行调用、嵌套调用、多轮交互调用
- **参数类型**：字符串、数字、布尔值、数组、嵌套对象等复杂类型

## 主要指标

- **Tool Selection Accuracy**（工具选择准确率）
- **Parameter Name Accuracy**（参数名正确率）
- **Parameter Value Accuracy**（参数值正确率，含类型检查和语义匹配）
- **Full Call Accuracy**（完整调用准确率）：工具名和所有参数均正确的比例
- BFCL 排行榜成绩（间接指标）

## 局限性

- 工具数量虽多（16K+），但以英文 API 为主，中文工具场景覆盖有限
- 部分工具调用的"正确答案"存在歧义（等效的不同参数组合）
- 过于关注精确匹配，可能低估能够通过近似调用完成任务的模型实际能力
