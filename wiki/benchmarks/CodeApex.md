# CodeApex

> 面向中文编程能力的综合评测基准，以 C++ 为核心语言，覆盖编程概念理解、代码补全和程序错误修复三大任务类型。

## 概述

CodeApex 由 Fu 等人于 2023 年发布，是专门针对中文编程教育场景设计的代码评测基准。与 HumanEval、MBPP 等以英文为主的代码基准不同，CodeApex 面向中文高校计算机专业学生，题目文本以中文描述为主，主要考察 C++ 语言（兼顾 C），反映中国大学编程课程的实际考核内容。

CodeApex 包含三种评测任务类型：其一是**编程概念理解**（conceptual understanding），以多选题形式测试模型对数据结构、算法、编程范式等概念的掌握；其二是**代码补全**（code completion），给定函数签名和注释，生成函数体实现；其三是**程序错误修复**（bug fixing），给定包含逻辑错误的代码片段，输出修正后的正确代码。这三种任务的组合使 CodeApex 能够较全面地评测模型的编程综合能力，而非仅测试代码生成。

评测结果揭示了主流 LLM 在 C++ 和中文代码场景下的显著短板：GPT-4 整体表现最佳，但在代码补全任务上仍有较大提升空间；中文开源模型的表现差异悬殊。CodeApex 为中文代码能力评测提供了一个标准化框架。

## 任务格式

- 格式：三任务混合（多选题 + 代码补全 + 错误修复）
- 主要语言：C++（兼顾 C）
- 题目语言：中文
- 评测方式：多选题准确率；代码任务使用测试用例执行验证（pass@1）

## 主要指标

- **准确率（Accuracy）**：概念理解任务（多选题）
- **pass@1**：代码补全和错误修复任务

## 局限性

- **C++ 单语言偏向**：以 C++ 为核心，对 Python、Java 等主流语言覆盖不足
- **中文场景局限**：题目描述和背景设定高度依赖中文教育场景，难以直接用于评测英文代码能力
- **题量有限**：各子任务题量相对较小，统计置信度受限
