# 事实性评测（Factuality Evaluation）

> 评测 LLM 生成内容与客观世界事实一致性的方法体系，是幻觉检测的核心任务，面临验证基础设施构建和自动化扩展的重大挑战。

## 定义

事实性评测（Factuality Evaluation）目标是：量化 LLM 生成文本中真实、准确的事实命题比例，识别错误的事实声明（幻觉）。

与简单的"正确性"评测不同，事实性评测需要处理：
- 开放域的任意事实声明（非选择题）
- 程度不同的错误（完全错误 vs 轻微不准确）
- 时间敏感的事实（历史事实 vs 可能已变化的当前事实）
- 无法验证的声明（主观判断、未来预测）

## 重要性（在 LLM 评测中）

事实性是 LLM 在高价值使用场景中最关键的能力维度：

1. **核心安全维度**：在医疗、法律、金融等领域，事实错误可能造成直接伤害
2. **RLHF 评测挑战**：人类标注者往往无法识别细微的事实错误，导致奖励模型对事实性不敏感
3. **LLM-as-judge 局限**：裁判 LLM 自身也可能有幻觉，无法可靠判断事实正确性
4. **知识截止与时效性**：评测模型对截止日期后事件的处理（应承认不知道而非虚构）

## 主要方法/实现

**TruthfulQA 方法**：
- 设计模仿常见错误信念的问题
- 要求模型既给出"真实性"判断也给出"信息量"判断
- 人工验证准确性

**FActScore（Min et al., 2023）**：
```
1. 让 LLM 生成关于主题的传记式段落
2. 将段落分解为原子事实（每个不可再分的命题）
3. 对每个原子事实，用 Wikipedia 等知识源验证
4. FActScore = 已验证的原子事实比例
```

**SAFE（Search-Augmented Factuality Evaluator）**：
- 用搜索引擎检索支持/反驳每个原子事实的证据
- 比静态知识库更能处理最新事实

**自动化评测工具**：
- GPT-4 作为事实核查器（有局限性）
- NLI 模型判断事实命题与知识库的蕴含关系
- 专用事实核查数据集（FEVER, FactCC）

## 局限与挑战

- **验证基础设施瓶颈**：可靠的外部知识源（Wikipedia、Wikidata）覆盖范围有限，专业领域事实无法自动验证
- **原子性分解的不一致**：不同系统对相同文本的原子事实分解结果不同，影响可比性
- **LLM 验证器的循环性**：用 LLM 验证 LLM 生成的事实，偏差可能相互掩盖
- **时间动态性**：正确答案可能随时间变化，评测需明确知识截止日期
- **灰色地带**：主观声明、价值判断、部分正确的陈述难以明确分类
