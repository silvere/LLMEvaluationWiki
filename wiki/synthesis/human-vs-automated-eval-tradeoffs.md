# 人工评测与自动评测的权衡

## 问题背景

"如何评测 LLM 的输出质量？"是 LLM 研究与应用中最核心、最难以解决的问题之一。人工评测和自动评测各有其不可替代的价值，也各有难以弥补的缺陷。实践中，从业者需要根据具体场景在两者之间做出权衡。

## 人工评测

### 优势

**最高效度（validity）**：对于开放式生成任务（创意写作、对话质量、有用性等），人类判断仍是最接近真实使用体验的评测标准。Chatbot Arena 等以人类投票为核心的排行榜与实际用户满意度高度相关。

**捕捉微妙差异**：人类评测者能识别语气适切性、文化敏感性、内容创意性等细微质量差异，这些维度难以被自动化指标捕捉。

### 劣势

**成本高**：众包平台（如 MTurk、Prolific）的评测费用通常为每条 $0.1-$1，大规模评测（数千至数万条）成本显著。专业评测者成本更高。

**速度慢**：收集到足够数量的可信人工标注需要数天到数周，无法在每次模型迭代时快速获得反馈。

**一致性差**：不同标注者对"质量"的定义存在主观差异。即使是同一标注者，在不同时段对相同内容的评分也可能不一致（标注者间一致性通常为 0.7-0.8 Kappa）。

**可扩展性差**：随着需要评测的模型数量增加，人工评测成本线性增长。

## 自动评测

### 优势

**可扩展、低成本**：自动评测可在几分钟内处理数千条样本，计算成本远低于人工。

**高可复现性**：固定的评测代码和参数确保不同时间、不同研究者运行得到相同结果。

**适合有标准答案的任务**：对于数学题、代码、事实性问答等有明确答案的任务，自动评测的精确性不亚于甚至优于人工。

### 劣势

**效度受限**：传统指标（BLEU、ROUGE）与人类判断的相关性较低，在开放式生成任务上尤为明显。

**LLM-as-Judge 的系统性偏差**：使用 GPT-4 等模型作为评判时，存在已知偏差——倾向于更长的回答、偏好与自身训练分布相似的风格。这些偏差难以完全消除。

**容易被"刷榜"**：当自动评测指标公开后，模型训练过程可能有意或无意地针对该指标优化，导致指标与真实质量脱钩（Goodhart 定律）。

## LLM-as-Judge：中间路径

LLM-as-Judge（使用强大 LLM 模拟人类评判）是近年兴起的折中方案：
- 成本约为纯人工评测的 1/100
- 速度接近自动评测
- MT-Bench 研究显示 GPT-4 评判与人类评判的一致性约为 80%

但它同时继承了两者的缺陷：成本高于传统自动指标，准确性不如真实人类判断，且存在系统性偏差。

## 实践中的选择框架

| 场景 | 推荐方案 |
|-----|---------|
| 模型快速迭代验证 | 自动评测（基于客观指标） |
| 用户体验/满意度评估 | 人工评测或 Chatbot Arena 投票 |
| 开放式生成质量 | LLM-as-Judge + 定期人工抽样验证 |
| 安全和有害内容检测 | 人工评测（高风险，需专业审核） |
| 生产监控 | 自动指标 + 异常时人工复核 |

## 趋势

随着 LLM 能力提升，LLM-as-Judge 的质量持续改善，但与人类评判的根本差距难以完全消除。多数顶级评测系统（如 Chatbot Arena）选择以人类投票为锚点，用自动评测快速筛选候选，用人工评测做最终验证的混合策略。
