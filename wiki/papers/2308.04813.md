# CLEVA: Chinese Language Models EVAluation Platform

> 构建面向中文 LLM 的综合评测平台 CLEVA，通过标准化提示流程、定期引入新数据和动态排行榜机制缓解数据污染问题。

## 核心主张
- [CLAIM]: 现有中文 LLM 评测缺乏统一的提示标准，不同评测间结果不可比；CLEVA 通过标准化的提示工作流和统一接口解决这一问题。
- [CLAIM]: 静态基准面临严重的数据污染风险——模型训练数据可能已包含测试集内容；CLEVA 通过持续引入新数据和每轮使用唯一子集来缓解污染。
- [CLAIM]: CLEVA 以极低的用户编码门槛（仅需鼠标操作和模型 API）支持对 23 个中文 LLM 的大规模评测，验证了平台的可用性和有效性。

## 方法论摘要

CLEVA 平台的核心设计理念是"可复现、可对比、抗污染"。在可复现层面，平台对所有任务统一定义提示模板、输出解析规则和评分方式，确保不同模型的评测结果在相同条件下产生。在抗污染层面，平台采用双重策略：一是定期生成新评测数据（而非依赖固定公开数据集），二是每轮排行榜使用随机采样的唯一子集，使得提前记忆测试数据变得困难。

平台覆盖多个评测维度，包括语言理解、知识问答、推理、代码生成、安全性等，并支持用户自定义评测任务。用户只需提供模型 API 接口，平台自动完成提示构建、模型调用、结果解析和得分计算全流程。

在验证实验中，研究团队对 23 个主流中文 LLM 进行了系统性评测，涵盖闭源商业模型（如 GPT 系列）和开源模型（如 ChatGLM 等），并公开了排行榜结果。

## 数据与结果

- 测试模型数量：23 个中文 LLM
- 评测任务覆盖：语言理解、知识问答、推理、代码、安全等多个维度
- 新数据比例：每轮引入显著比例的新生成/采集数据以缓解污染
- 用户门槛：仅需模型 API + 简单界面操作，无需额外编码

## 局限与争议

- 平台依赖模型 API 调用，对于不提供标准 API 的本地部署模型支持有限。
- 新数据的引入依赖人工或半自动化构建，数据质量和标注一致性有待持续保障。
- 排行榜机制的公平性依赖参与方诚实报告模型参数和训练数据，自我申报存在信息不对称风险。
- 平台主要聚焦中文，对中英双语混合场景或其他亚洲语言的支持有限。
